## 执行上下文
* 当我们调用一个函数时（激活），一个新的执行上下文就会被创建。`执行上下文可以理解为当前代码的执行环境`
* 一个`执行上下文`的生命周期可分为 `创建阶段` 和 `代码执行阶段` 两个阶段。

1. `创建阶段`：在这个阶段中，执行上下文会分别进行以下操作
* 创建 变量对象
* 建立 作用域链
* 确定 this 的指向
![执行上下文](./执行上下文.png)
2. `代码执行阶段`：创建完成之后，就会开始执行代码，并依次完成以下步骤
* 变量赋值
* 函数引用
* 执行其他代码
3. `销毁`
* 执行完毕，当前执行上下文出栈(call stack),等待被垃圾回收

* JavaScript 中的运行环境大概包括三种情况：
1. 全局环境：JavaScript 代码运行起来会首先进入该环境
2. 函数环境：当函数被调用执行时，会进入当前函数中执行代码
3. eval（不建议使用，可忽略）

* 因此在一个 JavaScript 程序中，很可能会产生`多个执行上下文`，而 JavaScript 引擎会以栈的方式来处理它们，
* 这个栈，我们称其为 `调用栈（Call Stack）`。`栈底永远都是全局上下文`，`而栈顶就是当前执行的上下文`。
* 处于栈顶的上下文执行完毕之后，就会自动出栈。

* `Call Stack有时候也被称为Execution Context Stack 执行上下文栈`
